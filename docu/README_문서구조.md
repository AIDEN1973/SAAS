# 📚 docu 폴더 문서 구조 가이드

## 문서 역할 및 관계

### 1. 사용자용 문서

#### `사용자_지시_가이드.md` (비기술자용)
- **역할**: 비기술자가 코드 생성 요청 시 참고하는 간단한 가이드
- **내용**: 3가지 템플릿, 사용 예시, 자동 검증 프로세스 요약
- **사용자**: 비기술자
- **목적**: 간단한 지시 방법 안내

---

### 2. AI용 문서

#### `코드생성_체크리스트.md` (AI 핵심 참조 문서) ⭐
- **역할**: AI가 코드 생성 시 **실제로 검증하는 상세 체크리스트**
- **내용**: 18개 항목의 구체적인 검증 항목
  - 아키텍처 레이어 확인
  - 의존성 경로 확인
  - Zero-Trust 원칙 확인
  - Service Layer 규칙 확인
  - 환경변수 접근 확인
  - 클라이언트-서버 분리 확인
  - 타입 안전성 확인
  - UI/UX 규칙 확인
  - 패키지 구조 확인
  - RLS 정책 확인
  - 의존성 방향 확인
  - API SDK 메서드 확인
  - Hook 패턴 확인
  - 에러 처리 확인
  - PII 마스킹 확인
  - 날짜/시간 처리 확인
  - SQL 인덱스 확인
  - 운영 규칙 확인
- **사용자**: AI (자동 참조)
- **목적**: 실제 검증 항목 확인

#### `AI_자동검증_프로세스.md` (AI 프로세스 가이드)
- **역할**: AI가 코드 생성 시 수행해야 하는 **검증 프로세스의 전체 흐름** 설명
- **내용**: 
  - 자동 검증 원칙
  - 검증 프로세스 (1단계~4단계)
  - 효율성 개선 방법
  - 주의사항
- **사용자**: AI (프로세스 참조)
- **목적**: 검증 프로세스 흐름 이해

---

  ↓

## 각 문서의 필요성

### ✅ `코드생성_체크리스트.md` - **필수**
- **이유**: AI가 실제로 검증할 때 참조하는 상세 체크리스트
- **대체 불가**: 다른 문서에는 구체적인 검증 항목이 없음
- **역할**: 18개 항목의 구체적인 확인 사항 제공

### ✅ `AI_자동검증_프로세스.md` - **필수**
- **이유**: 검증 프로세스의 전체 흐름을 설명
- **역할**: 언제, 어떻게 검증할지 프로세스 안내

### ✅ `사용자_지시_가이드.md` - **필수**
- **이유**: 비기술자가 쉽게 사용할 수 있는 가이드
- **역할**: 사용자 친화적인 지시 방법 제공
